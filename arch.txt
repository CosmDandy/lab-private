  Архитектурный аудит репозитория vpn-infra

  Текущее состояние: "всё в одном котле"

  Сейчас у тебя Terraform выполняет три роли одновременно: инфраструктура,
  конфигурация серверов (через cloud-init), и управление секретами. GitHub
  Actions выполняет ещё две: рендеринг конфигов и деплой. Ansible при этом
  используется только для домашних машин. В результате:

  - Cloud-init — одноразовая конфигурация, которую невозможно обновить без
  пересоздания сервера
  - 3 почти идентичных cloud-init шаблона (vpn.yaml.tftpl, mesh.yaml.tftpl,
  server.yaml.tftpl) с 90% дублирования
  - Нет Day-2 операций — обновить sysctl, Docker, SSH hardening можно только
  руками через SSH
  - Legacy код живёт рядом с новым, создавая путаницу

  Вопросы для интервью

  Прежде чем предлагать конкретный план реализации, мне нужно понять несколько
  вещей:

  1. Масштаб: Сколько серверов планируется в ближайшие 6 месяцев? Если 3-5,
  static inventory для Ansible достаточно. Если 10+, нужен dynamic inventory
  через hcloud plugin.
  2. Ansible запуск: Откуда планируешь запускать Ansible? Варианты:
    - Из devcontainer / Codespaces (нужен SSH-ключ)
    - Из GitHub Actions (нужен SSH-ключ в секретах + workflow)
    - С локальной машины
  3. Legacy миграция: Когда планируется полностью мигрировать vpn-hel-01 на
  Remnawave? Это определяет, стоит ли сейчас рефакторить legacy или просто
  пометить deprecated.
  4. Headscale vs Tailscale: Ты упомянул папку "Tailscale" — планируется переход
   с Headscale на Tailscale, или это просто общее название для mesh-слоя?
  5. Naming: ты предлагал templates/ переименовать. Из анализа получилось 3
  варианта:
    - stacks/ — чётко показывает, что это Docker Compose стеки
    - services/ — более общее
    - configs/ — но тогда конфликт с текущей legacy папкой

  Какой предпочитаешь?
  6. Приоритет: Что важнее сейчас — навести порядок в структуре (переименования,
   перемещения) или внедрить Ansible для серверной конфигурации? Или
  параллельно?
  7. Мониторинг: Планируется ли добавить мониторинг (Prometheus, node_exporter,
  alerts)? Это повлияет на структуру Ansible-ролей.
